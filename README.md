C SIMPLE SHELL

Simple Shell

A simple UNIX command interpreter written in C as part of the Holberton School curriculum.

Description:

This project implements a basic shell that reads commands from standard input and executes them. It mimics the behavior of sh /bin/sh with a limited set of features including built-in commands, executable path handling, and basic error management.

Requirements:

General

	∙	Allowed editors: vi, vim, emacs
	∙	All files will be compiled on Ubuntu 20.04 LTS using gcc, with the options -Wall -Werror -Wextra -pedantic -std=gnu89
	∙	All your files should end with a new line
	∙	A README.md file, at the root of the folder of the project, is mandatory
	∙	Your code should use the Betty style. It will be checked using betty-style.pl and betty-doc.pl
	∙	Your shell should not have any memory leaks
	∙	No more than 5 functions per file
	∙	You are not allowed to use the standard library. Any use of functions like printf, puts, etc., is forbidden
	∙	You are allowed to use _putchar
	∙	You don’t have to push _putchar.c, we will use our file. If you do it won’t be taken into account
	∙	In the following examples, the main.c files are shown as examples. You can use them to test your functions, but you don’t have to push them to your repo if you do we won’t take them into account
	∙	Our main files will include your header files and call all your functions
